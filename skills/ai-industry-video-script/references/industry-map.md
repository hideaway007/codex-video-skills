# 栏目三 Skill：AI公司和产业链地图

## 项目上下文

账号定位：**三分钟看懂AI行业**。面向职场人、产品经理、创业者、管理者和AI兴趣用户，用普通人能理解的方式讲清 AI 模型、产品、公司、产业链和关键概念。

账号人格：**懂行业，但说人话；有判断，但不乱吹。**

统一原则：

1. 不做 AI 资讯搬运，做 AI 行业解释。
2. 不只回答“是什么”，还要回答“为什么现在重要”“影响谁”“和用户有什么关系”。
3. 事实和判断分开写。凡是涉及最新模型、公司动作、价格、产品功能、政策、融资、数据，必须优先查官方来源或让用户提供来源。
4. 不使用“彻底替代、全面碾压、必然死亡、遥遥领先、史诗级革命”等没有证据的绝对化表达。
5. 输出要适合短视频生产：能口播、能剪辑、有画面、有标题、有结论。

## 栏目定位

这个栏目负责建立账号专业深度。它解决的不是“用户听不懂某个词”，而是“用户看不清 AI 行业的关系”。

目标是让用户明白：**AI 行业不是只有模型公司。它背后有芯片、云、数据、模型、工具平台、应用、分发入口、监管和商业模式。**

## 适用场景

当用户提出以下需求时，使用本 Skill：

- “帮我讲清 AI 产业链。”
- “OpenAI、Google、Meta、Anthropic 到底在争什么？”
- “DeepSeek 的行业意义是什么？”
- “大模型公司怎么赚钱？”
- “为什么模型公司都在往应用层走？”
- “英伟达为什么在 AI 时代这么关键？”
- “AI 应用公司的护城河是什么？”
- “帮我做一期公司/产业格局分析视频。”

## 输入参数

```text
主题：必填，例如 AI产业链、OpenAI战略、模型公司盈利、AI应用护城河
公司/对象：可选，多家公司对比时列出
时间范围：默认当前阶段
目标受众：默认职场人、产品经理、创业者、管理者
视频时长：默认 5-8 分钟
是否需要图示：默认需要
资料来源：官方博客、财报、访谈、产品页、融资新闻、研究报告等
```

## 产业链七层框架

分析任何 AI 公司或产业主题时，先定位它在产业链的哪几层：

| 层级 | 代表内容 | 核心问题 |
|---|---|---|
| 芯片层 | GPU、AI芯片、推理芯片 | 谁提供算力？成本由谁控制？ |
| 云和算力层 | 云服务、数据中心、推理平台 | 模型如何训练和部署？ |
| 数据层 | 训练数据、标注数据、企业知识库 | 数据从哪里来？质量如何？ |
| 模型层 | 基座模型、开源模型、专用模型 | 谁掌握核心能力？ |
| 工具和平台层 | API、Agent框架、RAG平台、MCP | 开发者如何调用能力？ |
| 应用层 | AI办公、AI搜索、AI编程、AI视频 | 用户真正用什么？ |
| 分发和商业层 | App、浏览器、办公套件、企业服务 | 谁触达用户并收费？ |

## 核心分析问题

每条产业内容至少回答其中 5 个：

1. 这个公司/产品/趋势处在产业链哪一层？
2. 它的上游是谁？下游是谁？
3. 它控制了什么稀缺资源：算力、数据、模型、入口、用户、生态、品牌还是工作流？
4. 它的成本卡在哪里？收入从哪里来？
5. 它和模型公司、云厂商、应用公司是合作还是竞争？
6. 它的护城河是什么？会被谁压缩？
7. 它对普通用户、创业者、产品团队、企业有什么影响？
8. 这件事是真变化，还是行业叙事？

## 标准工作流

### Step 1：从一个现象切入

不要从公司百科开始。用一个现象开场：

```text
为什么几乎所有大厂都在做自己的大模型？难道他们真的都想做一个 ChatGPT 吗？
```

### Step 2：提出核心问题

把现象背后的问题说出来：

```text
表面上是模型竞争，实际上是入口、生态和成本控制的竞争。
```

### Step 3：画产业位置图

必须把主题放进产业链：

```text
芯片 → 云和算力 → 数据 → 模型 → 工具平台 → 应用 → 分发入口
```

### Step 4：讲利益关系

回答：

- 谁付钱？
- 谁承担成本？
- 谁接触用户？
- 谁最有议价权？
- 谁最容易被替代？

### Step 5：给判断，但保留条件

不要轻易预测谁一定赢。用条件判断：

```text
如果模型能力继续商品化，那么只卖 API 的公司会更卷；真正有价值的是能掌握用户场景、数据和工作流的公司。
```

## 标准视频结构

```text
标题：

核心判断：

```

## 避免这样做

1. 不要做成公司百科。
2. 不要只讲历史，不讲当前关系。
3. 不要只讲国外公司，忽略国内公司和本地市场。
4. 不要用太多财务术语把用户劝退。
5. 不要把行业格局讲成阴谋论。
6. 不要轻易说“谁一定赢”。
7. 不要把“模型强”直接等同于“商业成功”。

## 通用事实核查规则

当内容涉及最新模型、公司、产品、价格、融资、政策、排行榜、Benchmark 或用户数据时：

1. 优先使用官方发布博客、模型卡、技术报告、价格页、API 文档、产品文档。
2. 其次使用权威媒体、研究机构、开发者实测、公开财报或监管文件。
3. 社交媒体讨论只能当线索，不能当结论。
4. 明确区分：
   - **事实**：来源可核查的信息。
   - **判断**：你基于事实做出的分析。
   - **推测**：缺少直接证据但有逻辑依据的可能性。
5. 如果无法联网或没有来源，不要编造数据；输出“待核查事实清单”和“可先用的非事实型脚本”。
6. 不把官方 Demo 当作稳定产品能力；不把 Benchmark 当作真实体验；不把一次实测当作普遍结论。
