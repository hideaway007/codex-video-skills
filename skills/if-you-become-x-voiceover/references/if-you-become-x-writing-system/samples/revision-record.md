# 样稿返修记录

本记录用于证明“生成 -> 评分 -> 返修”不是口头流程，而是可重复执行的校准动作。以下评分都是开头形态评估，不等于 7000-11000 字完整稿的最终 90% 判定。

## 返修总原则

- 保留类型机制：第二人称、幻想纠偏、低处落点、小胜转账单、系统压力。
- 不保留具体创作者声音：不使用专属自称、观众称呼、固定梗、私人经历口吻或可识别原句。
- 开头样稿的目标不是写完一生，而是在 1000 字左右快速完成“观众幻想 -> 你被投放 -> 规则开始收账”。

## 01 秦末戍边征发者

原始问题：

- 弱者处境清楚，但前 1000 字基本只有压力，缺少“第一次小胜”。
- 系统荒诞已经出现，但还可以让“有用也是账单”更早成立。

返修动作：

- 新增“你记住驿卒提醒，判断前方桥断，队伍少走死路”的小胜。
- 小胜后立刻转为责任：以后岔路、天气、病倒都要你参与判断。
- 把“有用不是奖励，有用是下一张账单”提前到开头尾部。

结果：

- 原始人工开头初评：82/100。
- 返修后人工开头预估：86/100。
- 下一步全稿重点：误期压力、大泽乡式主事件、从求活到不得不带头。

## 02 刚入行的短视频运营

原始问题：

- 职业祛魅和爆款账单明确。
- 人格变形只在评分备注里出现，正文中还没有提前发生。

返修动作：

- 新增“你开始用留存率、冲突点、二创空间审判现实生活”的段落。
- 把第一次爆款定义成“证据”：证明你做到过，所以之后做不到就叫退步。
- 保持行业机制在真实边界内：数据、标题、封面、评论区、复盘。

结果：

- 原始人工开头初评：86/100。
- 返修后人工开头预估：89/100。
- 下一步全稿重点：连续失败、账号崩盘、商业化压力、把人变成指标工具。

## 03 边境法师团新兵

原始问题：

- 荣耀耗材逻辑清楚，幻想拆解强。
- 大事件收账还停留在评分备注，没有在开头形成钩子。

返修动作：

- 新增“贵族监察官临时要求新兵演示”的权力压力。
- 新增“黑焰烽火打断演示”的大事件钩子。
- 把靶场命中从“天赋展示”改写成“可用性证明”。

结果：

- 原始人工开头初评：88/100。
- 返修后人工开头预估：90/100。
- 下一步全稿重点：战场第一次使用、敌方法术反制、从害怕爆开到接受自己是耗材。
