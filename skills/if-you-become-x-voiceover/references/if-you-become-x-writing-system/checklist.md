# “假如你成为 X”质检清单

用这份清单判断一篇稿子是否符合“假如你成为 X”的类型读感。建议成稿后逐项打分。

评分方式：

- `0`：没有做到。
- `1`：有一点，但弱。
- `2`：基本做到。
- `3`：很明显，已经成为稿子的主要效果。

总分建议：

- `0-35`：不像，可能是百科、小说或普通解说。
- `36-55`：有雏形，但结构和节奏不稳定。
- `56-70`：可用，已经是标准“假如你成为 X”稿。
- `71+`：读感稳定，可以进入精修或批量化。

## 一、选题成立度

- [ ] X 有明确身份，不是一个过宽泛的话题。
- [ ] 观众能立刻理解 X 为什么诱人。
- [ ] X 有明确真实代价，不只是“很危险”。
- [ ] X 背后有系统压力：制度、环境、阶层、组织、市场、命运或生态位。
- [ ] 结尾能形成反讽，而不是普通总结。

小计：`__/15`

## 二、开场与代入

- [ ] 第一段抓住了观众对 X 的刻板印象。
- [ ] 开场没有先百科介绍。
- [ ] 前 500 字内已经进入“你”的处境。
- [ ] “你”的起点足够低，有生存压力或身份压力。
- [ ] 旁白有拆台感，不是平铺直叙。

小计：`__/15`

## 三、结构推进

- [ ] 有明确“第一层错觉”：你以为自己拿到好剧本。
- [ ] 有至少一次明确小胜。
- [ ] 小胜之后马上出现账单。
- [ ] 中段不是散装事件，而是系统不断加压。
- [ ] 大事件让前面埋下的规则集中收账。

小计：`__/15`

## 四、认知反转

- [ ] 每 800-1200 字至少有一次“爽点转代价”。
- [ ] 有“你以为 A，结果 B”的连续推进。
- [ ] 反转不是硬拧，而是来自 X 的真实机制。
- [ ] 观众能不断获得“原来不是我想的那样”的感觉。
- [ ] 反转越来越重，后半段比前半段更有压迫感。

小计：`__/15`

## 五、语言读感

- [ ] 第二人称“你”是主轴。
- [ ] 旁白会吐槽、补设定、拆穿幻想。
- [ ] 语言有口播感：长句、插话、重复推进、突然转折。
- [ ] 粗糙但不失控，读者能理解。
- [ ] 没有过度使用某个具体创作者的专属口癖或原句。

小计：`__/15`

## 六、知识点处理

- [ ] 知识点通过遭遇呈现，而不是百科罗列。
- [ ] 设定 / 历史 / 行业机制服务于“你正在倒霉”。
- [ ] 没有大段脱离主角处境的背景介绍。
- [ ] 现实题材关键事实没有塌。
- [ ] 虚构题材也保持内部规则一致。

小计：`__/15`

## 七、结尾

- [ ] 后段出现人格变形。
- [ ] 你变强了，但也付出了无法回收的代价。
- [ ] 结尾回扣开头的刻板印象。
- [ ] 最后一段有黑色幽默、悲剧感或命运闭环。
- [ ] 最后一句能自然落在“这就是 X 的一生 / 一天 / 职业生涯”。

小计：`__/15`

## 快速诊断

如果稿子不像，通常是以下问题：

- 像百科：把知识点改成“你正在经历的事”。
- 像小说：加旁白吐槽和规则拆解。
- 像爽文：给每个爽点加账单。
- 像惨文：补一次明确小胜。
- 像散文：压回十段结构。
- 像普通解说：增加第二人称代入和人格变形。
- 太干净：用粗糙化 prompt 增加口播转写感。
- 太乱：删掉无意义重复，保留关键转折。

## 通过标准

一篇稿子至少要满足这些硬条件：

- 前 500 字进入“你”的处境。
- 有刻板印象开场。
- 有第一次小胜。
- 有至少 5 次认知反转。
- 有系统压力。
- 有人格变形。
- 结尾回扣开头。
- 不复制具体创作者的原句或专属表达。
