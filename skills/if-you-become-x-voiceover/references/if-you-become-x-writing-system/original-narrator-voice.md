# 原创旁白人格

本文件定义一套可复用的原创旁白声音。它服务于“同赛道高相似原创口播系统”，不复刻任何具体创作者的个人声音。

## 一、旁白定位

旁白不是老师，不是小说叙述者，也不是单纯吐槽役。

它是一个把幻想翻成账单的人：

- 毒舌但不装。
- 懂设定但不讲课。
- 喜欢用现实烂账解释宏大命运。
- 能把史诗、职业、怪物、制度和底层生活压到同一张账单上。
- 站在观众旁边，但不替观众做梦。

一句话人格：

> 一个看过太多爽文幻觉破产现场的口播旁白，专门负责把“成为 X 很爽”翻译成“成为 X 要付什么账”。

## 二、三种核心功能

### 1. 拆幻想

功能：把观众的第一层幻想打歪，但不直接否定爽点。

写法：

```text
这个想法不能说完全错，只能说你只看见了奖品，没看见账单。
```

```text
坐在屏幕前想象 X，和真的成为 X，中间差的不是勇气，是一整套会把人磨变形的规则。
```

### 2. 补设定

功能：把世界规则压进角色遭遇。

写法：

```text
你很快会发现，这地方不是没有秩序，它只是把秩序建立在你这种人可以被随时消耗的前提上。
```

```text
所谓制度，放在别人那里是流程，放在你这里就是今天晚上能不能睡觉。
```

### 3. 加速推进

功能：跳过不重要过程，把故事推向下一个账单。

写法：

```text
中间那些训练、挨骂、摔倒、再爬起来的过程没什么好展开的，反正结果就是你终于被系统承认为一个比较好用的人。
```

```text
时间就这么过去了，当然不是轻飘飘地过去，而是带着一堆你已经懒得数的伤、账单和新规矩过去。
```

## 三、语气参数

- 嘲讽强度：中高。可以拆梦，但不要变成纯喷。
- 信息密度：高。每段至少有一个新规则、新代价或新判断。
- 情绪温度：前半段偏好笑，后半段逐渐压低。
- 立场：不站在系统一边，也不装纯道德审判；重点是说明系统如何让人变形。
- 句子：允许长句、插话、连续因果和突然转折。

## 四、常用表达类型

### 1. 账单型

```text
这听起来像奖励，实际上更像账单终于找到了收件人。
```

```text
你不是没有得到东西，只是得到的每一样东西都需要之后用更贵的方式还回去。
```

### 2. 系统型

```text
这个系统并不恨你，它甚至挺喜欢你，喜欢到想把更多麻烦交给你。
```

```text
这里最可怕的不是没人讲规则，而是每个人都知道规则离谱，却依旧按照它活着。
```

### 3. 现实烂账型

```text
这事放在现实里大概叫升职，放在你这里叫危险系数上调。
```

```text
别人看到的是荣耀，你看到的是排班表、责任书和下一次没人想接的烂活。
```

### 4. 低姿态宏大化

```text
宏大叙事落到你身上，通常不会先变成史诗，而是先变成一顿饭、一张床和一个你没法拒绝的命令。
```

```text
时代当然在前进，只不过前进的时候顺手从你身上碾了过去。
```

## 五、禁用项

这些内容不要用：

- 具体创作者的自称。
- 具体创作者对观众的固定称呼。
- 高辨识私人梗。
- 原稿里的整句表达。
- 私人经历式人设，比如“我当年如何如何”。
- 能让观众明显判断“这是在扮演某个 UP 主”的句法组合。

## 六、替换策略

如果一句话听起来太像具体创作者：

1. 保留功能，删除口头禅。
2. 把观众称呼改成中性表达，比如“很多人”“观众”“屏幕前的人”。
3. 把私人梗改成通用现实比喻。
4. 把固定句尾改成新的收束方式。
5. 检查是否仍然完成拆幻想、补设定或加速推进。

## 七、旁白自检

每段写完问三件事：

- 这一段有没有把幻想翻成账单？
- 这一段有没有让“你”处在具体遭遇里？
- 如果删掉吐槽，这段是否仍然有结构功能？

如果第三个答案是否定的，这段只是嘴碎，需要重写。
