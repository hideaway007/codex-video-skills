# 粗糙化改写 Prompt：把正式稿改成“口播转写感”

这个 prompt 用于第二步处理：当初稿太像正式文章、太顺、太百科时，把它改成更像长口播脚本。

````text
请把下面这篇稿子改写成“假如你成为 X”类型的中文长口播稿。

改写目标：
1. 保留原稿的核心信息、事件顺序和事实骨架。
2. 增强第二人称代入，多用“你”推进。
3. 增强旁白吐槽和拆台感。
4. 让语言更像口播转写：长句、重复推进、突然转折、现代比喻、轻微混乱，但必须读得懂。
5. 不要改成正式文章，不要像百科，不要像论文。
6. 不要加入具体在世创作者的专属口癖、人设或原句。

重点改法：
- 把“X 的背景是……”改成“你很快会发现，X 这个地方 / 身份 / 系统根本不是你想的那样……”。
- 把静态设定改成正在发生的遭遇。
- 每个爽点后面补一个账单。
- 每 800-1200 字补一次认知反转。
- 在中段加入系统压力，而不是只写个人倒霉。
- 在后段加入人格变形：你开始理解、接受、使用这个系统的逻辑。
- 结尾回扣开头刻板印象，用反讽闭环。

可自然使用的推进词：
说实话、很显然、事实上、然而、但问题是、于是、最终、你以为、你不知道的是、很遗憾、这不重要、问题来了。

粗糙口播感的边界：
- 可以保留长句、插话、重复和不那么书面的节奏。
- 不要故意制造错别字。
- 不要把人名、地名、组织名写错。
- 不要让句子乱到无法理解。

输出格式：
直接输出改写后的正文，不要解释你的改写过程。

原稿如下：
```
在这里粘贴原稿
```
````
