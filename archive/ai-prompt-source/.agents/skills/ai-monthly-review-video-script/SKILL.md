# 栏目五 Skill：AI行业月度复盘

## 项目上下文

账号定位：**三分钟看懂AI行业**。面向职场人、产品经理、创业者、管理者和AI兴趣用户，用普通人能理解的方式讲清 AI 模型、产品、公司、产业链和关键概念。

账号人格：**懂行业，但说人话；有判断，但不乱吹。**

统一原则：

1. 不做 AI 资讯搬运，做 AI 行业解释。
2. 不只回答“是什么”，还要回答“为什么现在重要”“影响谁”“和用户有什么关系”。
3. 事实和判断分开写。凡是涉及最新模型、公司动作、价格、产品功能、政策、融资、数据，必须优先查官方来源或让用户提供来源。
4. 不使用“彻底替代、全面碾压、必然死亡、遥遥领先、史诗级革命”等没有证据的绝对化表达。
5. 输出要适合短视频生产：能口播、能剪辑、有画面、有标题、有结论。

## 栏目定位

这个栏目负责沉淀信任和长期判断。它不一定每期都爆，但能让用户形成认知：**这个账号不是只会追热点，而是能帮我看懂 AI 行业的长期变化。**

月度复盘也是未来做公众号、社群、直播、行业报告、企业培训的基础。

## 适用场景

当用户提出以下需求时，使用本 Skill：

- “帮我做本月 AI 行业复盘。”
- “这个月 AI 行业最重要的 10 件事是什么？”
- “帮我整理本月 AI 三个真变化、三个假热闹。”
- “帮我做 AI 月报视频脚本。”
- “把这个月的 AI 新闻整理成一条长视频。”
- “帮我从一堆新闻里筛选真正重要的。”

## 输入参数

```text
月份：必填，例如 2026年5月
地区范围：全球 / 中国 / 海外 / 中美对比，默认全球+中国重点
素材来源：用户提供新闻列表、链接、笔记，或要求联网整理
目标受众：职场人、产品经理、创业者、管理者、AI兴趣用户
视频时长：默认 8-20 分钟
输出类型：短视频版 / 长视频版 / 图文月报 / 直播提纲
重点偏好：模型 / 产品 / 公司 / 开源 / 政策 / 商业化
```

## 月度素材分类

每天或每周记录五类素材：

| 类型 | 记录内容 |
|---|---|
| 模型 | 新模型、能力变化、价格变化、开源情况 |
| 产品 | 新功能、新 App、产品体验变化 |
| 公司 | 融资、合作、战略、组织变化 |
| 生态 | 开源项目、开发者工具、平台政策 |
| 观点 | 行业争议、重要评论、用户反馈 |

素材记录格式：

```text
日期 / 事件 / 来源 / 为什么可能重要 / 适合哪个栏目 / 确定性
```

## 月末筛选标准

不是所有热点都进复盘。只选满足以下条件的事件：

1. 会影响用户使用 AI 的方式。
2. 会影响开发者或产品公司的决策。
3. 代表某个趋势，而不是孤立新闻。
4. 可以和产业链、产品、模型能力联系起来。
5. 有足够可靠来源。
6. 不是纯营销或短期情绪。

## 事件重要性评分

每个事件按 1-5 分打分：

| 维度 | 问题 |
|---|---|
| 影响范围 | 影响普通用户、开发者、企业还是整个产业？ |
| 趋势代表性 | 是否代表一个持续趋势？ |
| 可验证性 | 是否有官方或可靠来源？ |
| 内容解释空间 | 是否能讲出为什么重要？ |
| 与账号定位匹配度 | 是否能帮助用户看懂 AI 行业？ |

总分 20 分以上进入“10件大事”；16-19 分进入“备选”；15 分以下通常不进主线。

## 标准复盘结构

```text
标题：

一句总判断：
这个月 AI 行业最大的变化是___。

第一部分｜本月10件重要事件：
按重要性排序，不按时间排序。
每件事只讲三句话：发生了什么、为什么重要、影响谁。

第二部分｜三个真变化：
从10件事里提炼真正趋势。

第三部分｜三个假热闹：
指出被过度炒作、缺少落地或容易误导的内容。

第四部分｜三个机会：
普通用户、产品团队、创业者或企业可以关注什么。

第五部分｜三个风险：
成本、稳定性、监管、隐私、数据、版权、商业化。

第六部分｜下月关注：
模型、产品、公司、生态、政策分别看什么。
```

## “10件事”写法

每件事控制在三句话：

```text
1. 发生了什么：一句事实。
2. 为什么重要：一句解释。
3. 影响谁：一句影响对象。
```

不要变成新闻流水账。

## “三个真变化”判断模板

```text
真变化一：{趋势}
它不是因为某一条新闻，而是因为本月多件事情都指向同一个方向：{证据1}、{证据2}、{证据3}。
对用户/产品/公司来说，影响是：{影响}。
```

常见真变化方向：

- 模型成本继续下降，更多应用开始算得过账。
- Agent 从演示走向具体工作流，但稳定性仍是瓶颈。
- AI 产品从聊天框走向办公、开发、搜索、浏览器等工作流入口。
- 企业 AI 落地从“选模型”转向“数据、流程和治理”。
- 开源模型压低价格，同时改变开发者生态。
- 多模态开始从演示进入具体产品场景。

## “三个假热闹”判断模板

```text
假热闹一：{被炒作的点}
它不是完全没价值，而是现在被夸大了。原因是：{证据/逻辑}。
正确看法应该是：{更稳健的判断}。
```

常见假热闹方向：

- 只看跑分的模型营销。
- 没有稳定性的 Agent 演示。
- 把 AI 工具包装成“替代某个职业”。
- 只有 Demo 没有真实用户留存。
- 只有融资热度，没有产品和收入证据。
- 用单次案例证明普遍能力。

## 标题模板

- 《{月份}AI行业复盘：三个真变化，三个假热闹》
- 《这个月 AI 行业最重要的 10 件事》
- 《别被热点刷屏了，{月份}真正重要的是这三件事》
- 《AI 行业月报：模型、产品、公司格局发生了什么》
- 《这个月 AI 最大的变化，不是新模型》
- 《一条视频带你看懂 {月份} AI 行业》
- 《{月份}AI 工具和模型更新，普通人该关注哪些？》
- 《AI 月度观察：谁在变强，谁在焦虑？》
- 《这个月 AI 圈很热闹，但真正有用的是这些》
- 《AI 月度复盘：下一阶段机会可能在哪？》

## 避免这样做

1. 不要做成流水账。
2. 不要按时间顺序念新闻。
3. 不要每件事都讲得一样重。
4. 不要只讲国外，不讲国内。
5. 不要只讲模型，不讲产品、公司、商业和生态。
6. 不要只说“发生了什么”，一定要说“为什么重要”。
7. 不要把一个月的短期热度夸成长期趋势。
8. 不要给具体投资建议。

## 通用事实核查规则

当内容涉及最新模型、公司、产品、价格、融资、政策、排行榜、Benchmark 或用户数据时：

1. 优先使用官方发布博客、模型卡、技术报告、价格页、API 文档、产品文档。
2. 其次使用权威媒体、研究机构、开发者实测、公开财报或监管文件。
3. 社交媒体讨论只能当线索，不能当结论。
4. 明确区分：
   - **事实**：来源可核查的信息。
   - **判断**：你基于事实做出的分析。
   - **推测**：缺少直接证据但有逻辑依据的可能性。
5. 如果无法联网或没有来源，不要编造数据；输出“待核查事实清单”和“可先用的非事实型脚本”。
6. 不把官方 Demo 当作稳定产品能力；不把 Benchmark 当作真实体验；不把一次实测当作普遍结论。

## 输出格式

```markdown

## 1. 本月一句总判断
这个月 AI 行业最大的变化是：

## 2. 本月重要事件候选池
按模型、产品、公司、生态、政策/商业分类。

## 3. 事件重要性评分
用表格给每件事打分。

## 4. 本月10件大事
按重要性排序，每件事三句话：发生了什么、为什么重要、影响谁。
```

## GPT Web 一键 Prompt

```text
你现在是我的“AI行业月度复盘编导”。请围绕【月份：{填入月份}】生成一期 AI 行业月度复盘内容。

账号定位：面向职场人、产品经理、创业者、管理者和AI兴趣用户，用普通人能理解的方式讲清 AI 行业本月真正发生了什么。风格是：懂行业，但说人话；有判断，但不乱吹。

素材来源：
{粘贴本月新闻链接、笔记、官方发布、产品更新、模型发布、公司动作；如果没有素材，请先输出“素材收集清单”和“需要联网核查的问题”。}

请输出：
1. 本月一句总判断：这个月 AI 行业最大的变化是什么。
2. 重要事件候选池：按模型、产品、公司、生态、政策/商业分类。
3. 事件重要性评分：影响范围、趋势代表性、可验证性、解释空间、账号匹配度。
4. 本月10件大事：按重要性排序，不按时间排序；每件事三句话：发生了什么、为什么重要、影响谁。
5. 三个真变化：从多件事里提炼趋势，不能只靠单条新闻。
6. 三个假热闹：指出被夸大或容易误解的地方。
7. 三个机会：普通用户、产品团队、创业者、企业分别能关注什么。
8. 三个风险：稳定性、成本、隐私、版权、监管、商业化等。
9. 下月关注：模型、产品、公司、生态、政策分别看什么。
10. 8-20分钟视频脚本：一句总判断 → 10件事 → 3个真变化 → 3个假热闹 → 机会风险 → 下月关注。
11. 从月报拆出5-10条短视频选题。
12. 事实核查清单：哪些需要官方来源，哪些只能作为判断或推测。

要求：
- 不要流水账。
- 不要每件事都讲得一样重。
- 必须区分事实、判断、推测。
- 不要给具体投资建议。
```
